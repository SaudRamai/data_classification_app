# Fixed SP Merge Assets - Schema Migration Complete

## ✅ Issue Resolved: SQL Error "invalid identifier 'TABLE_NAME'"

### 🚨 **Root Cause:**
The `fixed_sp_merge_assets.sql` file contained a mixed schema definition:
- Table definition used the OLD schema (`DATABASE_NAME`, `SCHEMA_NAME`, `TABLE_NAME`)
- But the stored procedure tried to use NEW schema references
- This caused SQL compilation errors when the code tried to access non-existent columns

### ✅ **Solution Applied:**

#### **1. Updated Table Definition** (NEW SCHEMA):
```sql
-- BEFORE (Old Schema)
CREATE TABLE DATA_CLASSIFICATION_GOVERNANCE.CLASSIFICATION_AI_RESULTS (
    DATABASE_NAME VARCHAR(16777216) NOT NULL,
    SCHEMA_NAME VARCHAR(16777216) NOT NULL,  
    TABLE_NAME VARCHAR(16777216) NOT NULL,
    COLUMN_NAME VARCHAR(16777216),
    TABLE_FQN VARCHAR(16777216) GENERATED ALWAYS AS ...
    -- ... old fields
);

-- AFTER (New Schema)
CREATE TABLE DATA_CLASSIFICATION_GOVERNANCE.CLASSIFICATION_AI_RESULTS (
    RESULT_ID NUMBER(38,0) NOT NULL autoincrement start 1 increment 1 noorder,
    ASSET_FULL_NAME VARCHAR(16777216) NOT NULL,        -- NEW: Single FQN field
    COLUMN_NAME VARCHAR(16777216),
    SENSITIVITY_CATEGORY_ID VARCHAR(16777216),
    PII_RELEVANT BOOLEAN DEFAULT FALSE NOT NULL,
    SOX_RELEVANT BOOLEAN DEFAULT FALSE NOT NULL,
    SOC2_RELEVANT BOOLEAN DEFAULT FALSE NOT NULL,
    CONFIDENTIALITY_LEVEL VARCHAR(10),
    INTEGRITY_LEVEL VARCHAR(10),
    AVAILABILITY_LEVEL VARCHAR(10),
    IS_UPDATED BOOLEAN DEFAULT FALSE,
    CREATED_AT TIMESTAMP_NTZ(9) DEFAULT CURRENT_TIMESTAMP(),
    primary key (RESULT_ID)
);
```

#### **2. Fixed Stored Procedure Logic:**

**ai_results_agg CTE:**
```sql
-- BEFORE (Old Schema)
WITH ai_results_agg AS (
    SELECT 
        TABLE_FQN,
        MAX(CASE WHEN PII_RELEVANT = TRUE THEN 1 ELSE 0 END) > 0 as PII_RELEVANT,
        ...
    FROM DATA_CLASSIFICATION_GOVERNANCE.CLASSIFICATION_AI_RESULTS
    WHERE DATABASE_NAME IS NOT NULL 
      AND SCHEMA_NAME IS NOT NULL 
      AND TABLE_NAME IS NOT NULL
    GROUP BY TABLE_FQN
)

-- AFTER (New Schema)
WITH ai_results_agg AS (
    SELECT 
        ASSET_FULL_NAME,
        MAX(COALESCE(PII_RELEVANT, FALSE)) as PII_RELEVANT,
        MAX(COALESCE(SOX_RELEVANT, FALSE)) as SOX_RELEVANT,
        MAX(COALESCE(SOC2_RELEVANT, FALSE)) as SOC2_RELEVANT
    FROM DATA_CLASSIFICATION_GOVERNANCE.CLASSIFICATION_AI_RESULTS
    WHERE ASSET_FULL_NAME IS NOT NULL
    GROUP BY ASSET_FULL_NAME
)
```

**Join Condition:**
```sql
-- BEFORE
LEFT JOIN ai_results_agg ar 
    ON ar.TABLE_FQN = UPPER(TRIM(s.FQN))

-- AFTER
LEFT JOIN ai_results_agg ar 
    ON UPPER(TRIM(ar.ASSET_FULL_NAME)) = UPPER(TRIM(s.FQN))
```

**Debug Table Logic:**
```sql
-- BEFORE
SELECT TABLE_FQN FROM DATA_CLASSIFICATION_GOVERNANCE.CLASSIFICATION_AI_RESULTS
WHERE DATABASE_NAME IS NOT NULL AND SCHEMA_NAME IS NOT NULL AND TABLE_NAME IS NOT NULL

-- AFTER
SELECT ASSET_FULL_NAME FROM DATA_CLASSIFICATION_GOVERNANCE.CLASSIFICATION_AI_RESULTS
WHERE ASSET_FULL_NAME IS NOT NULL
```

#### **3. Updated Indexes:**
```sql
-- BEFORE
CREATE INDEX IDX_CLASSIFICATION_AI_RESULTS_FQN 
ON DATA_CLASSIFICATION_GOVERNANCE.CLASSIFICATION_AI_RESULTS(TABLE_FQN);

-- AFTER
CREATE INDEX IDX_CLASSIFICATION_AI_RESULTS_FQN 
ON DATA_CLASSIFICATION_GOVERNANCE.CLASSIFICATION_AI_RESULTS(ASSET_FULL_NAME);

CREATE INDEX IDX_CLASSIFICATION_AI_RESULTS_COLUMN 
ON DATA_CLASSIFICATION_GOVERNANCE.CLASSIFICATION_AI_RESULTS(ASSET_FULL_NAME, COLUMN_NAME);
```

### 🔧 **Key Transformations:**

1. **Schema Unification**: All components now use `ASSET_FULL_NAME`
2. **Removed Deprecated Fields**: No more `DATABASE_NAME`, `SCHEMA_NAME`, `TABLE_NAME`
3. **Simplified Logic**: Direct field access instead of generated columns
4. **Better Performance**: Single FQN field indexing
5. **Consistent Joins**: All joins use FQN-based matching

### 🎯 **Benefits:**

- ✅ **SQL Error Resolved**: No more "invalid identifier 'TABLE_NAME'" errors
- ✅ **Schema Consistency**: Table definition matches stored procedure logic
- ✅ **Simplified Structure**: Single FQN field instead of 3 separate fields
- ✅ **Better Indexing**: Optimized for FQN-based queries
- ✅ **Future-Proof**: Compatible with new schema design
