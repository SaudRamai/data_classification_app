# SQL Error Fix - Invalid Identifier 'TABLE_NAME'

## 🚨 Error Description
```
Failed to fetch inventory view: 000904 (42000): SQL compilation error: 
error line 39 at position 111 invalid identifier 'TABLE_NAME'
```

## 🔍 Root Cause
The `_fetch_classification_history()` function was trying to reference `TABLE_NAME` in a fallback SQL query, but the new `CLASSIFICATION_AI_RESULTS` schema doesn't have this column.

## ✅ Fix Applied

### Before (Problematic Code):
```python
# Build query dynamically based on available columns
asset_fqn_col = "r.ASSET_FULL_NAME" if 'ASSET_FULL_NAME' in col_names else "CONCAT(COALESCE(r.SCHEMA_NAME, 'UNKNOWN'), '.', COALESCE(r.TABLE_NAME, 'UNKNOWN')) as ASSET_FULL_NAME"
```

### After (Fixed Code):
```python
# Build query dynamically based on available columns
if 'ASSET_FULL_NAME' in col_names:
    asset_fqn_col = "r.ASSET_FULL_NAME"
elif 'DATABASE_NAME' in col_names and 'SCHEMA_NAME' in col_names and 'TABLE_NAME' in col_names:
    asset_fqn_col = "CONCAT(COALESCE(r.DATABASE_NAME, 'UNKNOWN'), '.', COALESCE(r.SCHEMA_NAME, 'UNKNOWN'), '.', COALESCE(r.TABLE_NAME, 'UNKNOWN')) as ASSET_FULL_NAME"
else:
    # Fallback for completely missing FQN components
    asset_fqn_col = "'UNKNOWN.UNKNOWN.UNKNOWN' as ASSET_FULL_NAME"
    logger.warning("ASSET_FULL_NAME and component columns not found. Using placeholder.")
```
