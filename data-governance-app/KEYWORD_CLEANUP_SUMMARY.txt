================================================================================
COMPLIANCE ALIGNMENT
================================================================================

GDPR (General Data Protection Regulation):
- Focus on data that directly identifies individuals
- SSN, passport, biometrics, health data

SOX (Sarbanes-Oxley Act):
- Focus on financial reporting and controls
- Bank accounts, credit cards, financial transactions

SOC2 (Service Organization Control 2):
- Focus on system access and security
- Passwords, API keys, tokens, encrypted data

HIPAA (Health Insurance Portability and Accountability Act):
- Protected Health Information (PHI)
- Health condition data

PCI-DSS (Payment Card Industry Data Security Standard):
- Payment card data
- Credit card numbers, CVV, expiry dates

================================================================================
CLASSIFICATION CRITERIA
================================================================================

‚úÖ CRITICAL (Kept) - Data that:
-------------------------------
- Directly identifies individuals (SSN, passport, biometrics)
- Provides direct financial access (bank accounts, credit cards)
- Grants system access (passwords, API keys, tokens)
- Cannot be easily changed (biometrics, government IDs)
- Creates immediate risk if exposed (fraud, identity theft)

‚ùå LOWER RISK (Removed) - Data that:
-------------------------------------
- Is a generic identifier (customer IDs)
- Is publicly available (routing numbers, SWIFT codes)
- Is context-dependent (IP addresses, device IDs)
- Is duplicative (two-factor phone/email)
- Is a reference ID (agreement IDs, question text)
- Cannot be used alone to cause harm
