# User Guide: Snowflake Data Governance & Classification App

## 1. Introduction

Welcome to the **Snowflake Data Governance Application**. This Native App helps organizations discover, classify, and protect their sensitive data residing in Snowflake. It provides a unified interface for Data Stewards, Compliance Officers, and Administrators to manage data privacy and regulatory requirements.

## 2. Getting Started

### Accessing the App
The application is accessible directly within your Snowflake account under the "Apps" section.
- **Login**: Use your standard Snowflake credentials.
- **Roles**: Your view and capabilities depend on your assigned role (e.g., `DATA_STEWARD`, `APP_ADMIN`, or `VIEWER`).

## 3. Application Modules

### 3.1 Data Assets Inventory (`1_Data_Assets`)
**Purpose**: View a comprehensive catalog of all tables and views monitored by the application.
- **Browse**: Filter by Database, Schema, or classification tags.
- **PII Detection**: Quickly identify columns flagged as containing Personally Identifiable Information (PII).
- **Search**: Use the search bar to find specific assets by name.

### 3.2 Executive Dashboard (`2_Dashboard`)
**Purpose**: High-level overview of the organization's data governance posture.
- **Key Metrics**: View total classified assets, sensitive data exposure, and compliance health scores.
- **Trends**: Track classification progress over time.
- **Risk Heatmap**: Visualize areas with high concentrations of sensitive data.

### 3.3 Classification Workflow (`3_Classification`)
**Purpose**: The primary workspace for Data Stewards to review and operationalize data classification.
- **Task Queue**: Review pending classification recommendations generated by the automated scanner.
- **Actions**:
    - **Approve**: Accept the AI-recommended tags.
    - **Reject/Edit**: Manually override recommendations.
    - **Escalate**: Flag complex cases for senior review.
- **My Tasks**: Track your personal queue of assigned reviews.

### 3.4 Compliance Reporting (`4_Compliance`)
**Purpose**: Generate reports for regulatory frameworks (GDPR, CCPA, HIPAA, etc.).
- **Impact Analysis**: See which datasets fall under specific regulations.
- **Export**: Download reports as PDF or Excel for external audits.

### 3.5 Data Intelligence (`5_Data_Intelligence`)
**Purpose**: Advanced analytics and AI-driven insights.
- **Pattern Recognition**: View detected patterns in data (e.g., credit card numbers, email addresses).
- **Quality Checks**: Monitor data quality metrics alongside classification.

### 3.6 Policy & Guidance (`6_Policy_Guidance`)
**Purpose**: Reference for internal data governance policies.
- **Definitions**: Understand what constitutes "Confidential", "Internal", or "Public" data.
- **Handling Guidelines**: Instructions on how to handle different categories of data.

### 3.7 System Configuration (`7_System_Configuration`)
**Purpose**: Admin-only settings for the application.
- **Scanning Rules**: Configure regex patterns and sensitivity thresholds.
- **Integration**: Manage connections to external systems (e.g., Jira, Slack).
- **User Management**: Assign roles and permissions.

## 4. Key Concepts

### Automated PII Scanning
The app runs a background process using Snowflake's `EXTRACT_SEMANTIC_CATEGORIES` to automatically scan your data. It assigns a "Confidence Score" to each finding. High-confidence findings may be auto-tagged, while lower-confidence findings generate manual review tasks.

### Classification Tags
Classification tags are applied directly to Snowflake objects using *Snowflake Tags*. This ensures that the classification travels with the data, even if it is queried outside of this application.

## 5. FAQ

**Q: How often is the data scanned?**
A: Scanning schedules are defined by your Administrator in the System Configuration tab. typically, it runs nightly or upon new table creation.

**Q: Can I export the list of sensitive assets?**
A: Yes, navigation to the **Compliance** tab allows for exporting detailed asset inventories.
